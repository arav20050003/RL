## 🚀 About the Project

Traditional supply chain management relies on rigid mathematical formulas (like the `(s,Q)` policy). While effective under minor volatility, these systems structurally shatter under significant supply chain shocks (e.g., severe natural disasters, port strikes, geopolitical events) because they *react* rather than *anticipate*.

This thesis project investigates if **LLM-Augmented Reinforcement Learning (RL)** can solve this. We test whether an RL agent that is continuously fed a probabilistic "risk score"—generated by an LLM parsing real-time news headlines—can avoid the catastrophic over-hedging typical of standard disruption-aware models. 

### The Core Discovery
We mathematically proved `(t = 5.43, p < 0.0001)` that feeding an AI a strict **binary** flag (e.g., `Disaster = True`) causes it to panic and crash its profitability (a 61% collapse). However, by utilizing our **LLM Oracle News Module** to grant the agent nuanced, noisy semantic understanding, the agent completely avoids the trap and navigates the disruption resiliently.

## 🏗 System Architecture

The project is executed in a two-phase architecture:

1. **Phase 1: Environment & Baseline Training**
   - **`ScimaiEnv`**: A highly rigorous Gymnasium environment simulating a 1-Product, 1-Warehouse supply chain.
   - **Baselines**: Proximal Policy Optimization (PPO) and Advantage Actor-Critic (A2C).
   - *Goal*: Successfully replicate the Stranieri & Stella (2023) performance benchmarks before attempting novel research.

2. **Phase 2: Disruption & LLM Oracle (Novel Contribution)**
   - **Disruption Generator**: Synthesizes market chaos using specific regimes (*Frequent-Short*, *Infrequent-Long*, *Stress Test*).
   - **Oracle News Module**: Instead of using expensive LLM API calls during training, a sophisticated rule-based pipeline simulates an LLM extracting continuous risk scores `[0, 1]` from news headlines (e.g., "SEVERE HURRICANE WARNING").

## 📊 Empirical Results

### Phase 1: Replication Validation (Stranieri & Stella 2023)
The economic framework of our simulator perfectly matched the original paper.
- **Oracle Model:** 1.3% Delta 
- **A2C Algorithm:** 4.7% Delta 
- **PPO Algorithm:** 32% Outperformance *(Our modernized SB3 architecture heavily outperformed their Ray/RLlib 1.5.2)*

### Phase 2: Core Discovery (Frequent-Short Regime)
*Results over 100 episodes:*
- **PPO Blind:** 1,031 Avg Profit
- **Disrupt-Aware PPO:** 398 Avg Profit *(Triggered 61% Over-Hedging Collapse)*
- **LLM-Augmented PPO:** 984 Avg Profit *(Zero Collapse)*

## 🛠 Getting Started

### 1. Installation

Clone the repo and install the Python backend dependencies:
```bash
git clone https://github.com/your-username/inventory_rl_thesis.git
cd inventory_rl_thesis
pip install -r requirements.txt
```

### 2. Running the RL Pipeline

You can run the full pipeline to train the agents, test the disruptions, execute the T-tests, and generate the plots:

```bash
# Run both Phase 1 and Phase 2 training and evaluation
python3 main.py --phase both

# (Alternatively) If models are already trained, output the analysis instantly
python3 main.py --phase both --eval-only
```

### 3. Launching the 3D Dashboard

This repository features an "Apple-level" interactive 3D frontend to explore the thesis findings dynamically.

```bash
cd website
npm install
npm run dev
```
Open `http://localhost:5173` to explore the 3D laboratory environment overlaying our detailed methodology and empirical charts.
